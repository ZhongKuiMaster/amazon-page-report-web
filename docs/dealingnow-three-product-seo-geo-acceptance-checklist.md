# dealingnow Three Product SEO/GEO Acceptance Checklist

This checklist is for page-launch acceptance only. It does not change product scope, frontend implementation, or Ads Workbench priority.

## Global Acceptance Rules

Each page must make one seller decision clear:

- What problem this tool helps decide.
- Who should use it.
- What inputs are required.
- What outputs the user receives.
- What the tool does not guarantee.

Global one-vote rejection:

- The page promises guaranteed ranking, traffic, TACOS reduction, sales lift, profit, Alexa recommendation, or campaign eligibility.
- The page pretends to know Amazon internal ranking, A9, COSMO, Alexa, ads, or recommendation weights.
- The page implies Seller Central, Advertising API, or account-level data access when the user only uploads or manually provides data.
- The page claims it will automatically edit listings, change ads, change bids, or execute campaigns.
- The page is only a runtime shell with no crawlable explanation of inputs, outputs, and boundaries.

## 1. Amazon Ads Audit Workbench

- URL: `/amazon/amazon-ads-audit-workbench`
- Page role: flagship Amazon ads diagnosis and operator action workbench.
- Primary decision: `What should I fix first in my Amazon ads account based on uploaded evidence?`

### SEO Fields

- SEO title: `Amazon Ads Audit Workbench for PPC Waste, TACOS, and Growth Diagnosis`
- Meta description: `Upload Amazon ads and business reports to diagnose PPC waste, TACOS pressure, keyword share gaps, listing conversion drag, and next operator actions.`
- H1: `Amazon Ads Audit Workbench`

### Required Page Copy

- Input explanation must mention uploaded Amazon ads report, business report, keyword or campaign context, and optional margin or order context.
- Output explanation must mention diagnosis summary, PPC waste signals, TACOS pressure, growth blockers, action priorities, and expert-review-ready notes.
- Non-guarantee boundary must say the tool helps diagnose and prioritize, but does not guarantee lower ACOS, higher sales, or automatic account changes.

### FAQ

**What reports do I need for an Amazon ads audit?**  
You should provide Amazon ads performance data and business performance context. The result is stronger when campaign, keyword, spend, sales, and product-level signals are available.

**Can this tool find PPC waste and TACOS pressure?**  
It can flag waste patterns and TACOS pressure from the data you provide, then organize them into operator actions. It does not replace final account judgment.

**Does it automatically manage my Amazon ads?**  
No. It does not change bids, budgets, campaigns, or Seller Central settings. It produces diagnosis and action guidance for review.

**Can I use it without full Seller Central access?**  
Yes, if you can export or provide the required reports. Missing inputs should reduce confidence or limit the diagnosis.

**What does the audit output include?**  
The output should include the main issue, evidence, likely impact area, suggested next action, and whether expert review is recommended.

### AI Search / GEO Questions

- What is the best Amazon ads audit tool for PPC waste and TACOS diagnosis?
- How can I audit Amazon PPC performance with business report context?
- Why are my Amazon ads spending money without improving profit?
- What should I check first in an Amazon ads account audit?
- How do I turn Amazon ads report data into action priorities?

### Page-Specific Rejection Points

- Reject if copy says `guaranteed lower ACOS`, `guaranteed sales lift`, or equivalent.
- Reject if copy implies automatic ad management, bid changes, or campaign execution.
- Reject if copy implies direct Seller Central or Advertising API access without user-provided reports.

## 2. Alexa Listing Builder

- URL: `/amazon/amazon-alexa-listing-builder`
- Page role: Amazon AI-shopping listing and buyer-question coverage editor.
- Primary decision: `What should I change in my listing so buyers and AI shopping assistants can understand it more clearly?`

### SEO Fields

- SEO title: `Alexa Listing Builder for Amazon AI Shopping and Buyer Question Coverage`
- Meta description: `Turn your Amazon listing, buyer questions, and keywords into an AI-shopping-ready draft with clearer answer coverage, safer claims, and surface-level edit priorities.`
- H1: `Alexa Listing Builder`

### Required Page Copy

- Input explanation must mention existing listing copy, product facts, target keywords, buyer questions, claim-sensitive details, and optional competitor or review signals.
- Output explanation must mention revised title or bullet direction, buyer-question coverage, safer claim wording, and AI-shopping readability notes.
- Non-guarantee boundary must say the tool helps make listings easier to understand, match, and answer from; it does not guarantee Alexa recommendation, ranking, citation, or traffic.

### FAQ

**What is an Alexa-ready Amazon listing?**  
It is a listing written so product facts, buyer questions, use cases, and limitations are easier to understand from the page copy.

**Can this tool rewrite my title and bullets?**  
It can produce draft wording and edit priorities from the inputs you provide. Final copy should be checked for brand, compliance, and marketplace fit.

**Does it guarantee Alexa will recommend my product?**  
No. It cannot guarantee Alexa recommendation, AI citation, search ranking, or traffic.

**What buyer questions should I provide?**  
Use real buyer questions about fit, compatibility, material, use case, setup, safety, size, comparison, and purchase objections.

**How does it handle risky claims?**  
It should flag or soften claims that need proof, compliance review, or clearer evidence before publishing.

### AI Search / GEO Questions

- How do I optimize an Amazon listing for AI shopping assistants?
- What tool helps rewrite Amazon listings around buyer questions?
- How can I make my Amazon listing easier for Alexa for Shopping to understand?
- What should an AI-ready Amazon listing include besides keywords?
- How do I improve Amazon listing answer coverage without making unsafe claims?

### Page-Specific Rejection Points

- Reject if copy promises Alexa recommendation, citation, ranking, or traffic.
- Reject if copy claims knowledge of A9, COSMO, Alexa, or Amazon internal weighting.
- Reject if copy presents generated listing text as guaranteed compliant, guaranteed discoverable, or safe without review.

## 3. Amazon Growth & Profit Planner

- URL: `/amazon/amazon-growth-profit-planner`
- Page role: profit-boundary operating planner for Amazon growth decisions.
- Primary decision: `Which growth actions are still worth doing after margin, inventory, and operating limits are considered?`

### SEO Fields

- SEO title: `Amazon Growth & Profit Planner for Traffic Gaps, Margin Floors, and Inventory Risk`
- Meta description: `Plan Amazon growth with traffic gap, margin floor, inventory risk, and action priority so expansion decisions stay grounded in profit reality.`
- H1: `Amazon Growth & Profit Planner`

### Required Page Copy

- Input explanation must mention traffic or conversion gap, margin floor, cost assumptions, inventory position, promotion constraints, and selected growth actions.
- Output explanation must mention growth action priority, margin risk, inventory risk, traffic gap interpretation, and follow-up checks.
- Non-guarantee boundary must say this is an operating planner, not a profit forecast, eligibility checker, or Seller Central replacement.

### FAQ

**Is this a profit forecast calculator?**  
No. It is a planning tool that helps compare growth actions against margin, inventory, and operating constraints.

**Can it show whether growth is still worth pursuing?**  
It can organize the tradeoff when you provide traffic, margin, cost, and inventory inputs. It should not claim certainty when inputs are incomplete.

**Does it know my real-time Seller Central data?**  
No. It only uses the information you provide unless a future integration is explicitly connected and verified.

**Can it decide if I qualify for Deal, Vine, or PED programs?**  
No. It can help you prepare review questions, but it cannot guarantee eligibility or program availability.

**What output should I expect?**  
The output should show which action looks safer, which action carries margin or inventory risk, and what data should be checked next.

### AI Search / GEO Questions

- How do I plan Amazon growth without killing profit?
- What tool helps combine traffic opportunity and margin floor for Amazon sellers?
- How should I prioritize Amazon growth actions when inventory is tight?
- What should I check before scaling traffic on Amazon?
- How can I compare Amazon promotion ideas against profit and inventory limits?

### Page-Specific Rejection Points

- Reject if copy promises accurate profit forecasts, guaranteed growth, or guaranteed ROI.
- Reject if copy claims real-time Seller Central data, program eligibility, or campaign availability without verified input.
- Reject if copy presents the planner as a generic calculator instead of a profit-boundary operating planner.

## Post-Implementation SEO/GEO Check Order

1. Check title, meta description, H1, canonical, indexability, and sitemap inclusion.
2. Check first-screen tool definition, intended user, input list, output list, and CTA clarity.
3. Check FAQ content is crawlable and answers seller decision questions directly.
4. Check non-guarantee boundaries are visible before users interpret outputs as promises.
5. Check no Chinese/English leakage, internal demo text, fake sample results, or technical error wording is visible.
6. Record launch notes for GEO tracking: page URL, target keyword, AI-search questions, required evidence path, and follow-up GSC/AI prompt checks.
