# dealingnow Amazon Three Tools SEO/GEO Brief

## 1. Core Positioning

`dealingnow.com` should not look like a thin Amazon tool directory. It should read as an Amazon operator tool system with three high-value expert workbenches:

1. `Amazon Ads Audit Workbench`
   - Primary role: flagship expert audit tool.
   - SEO/GEO positioning: diagnose Amazon ads waste, growth blockers, and next actions from real report inputs.
   - User decision supported: `What should I fix first in my Amazon ads account based on evidence, not guesswork?`

2. `Alexa Listing Builder`
   - Primary role: AI-shopping listing optimization tool.
   - SEO/GEO positioning: help sellers rewrite listing surfaces so Amazon AI shopping systems can better understand, match, and cite the product.
   - User decision supported: `What should I change in my listing first so buyer questions are answered more clearly without making unsafe claims?`

3. `Amazon Growth & Profit Planner`
   - Primary role: growth planning and profit-boundary tool.
   - SEO/GEO positioning: turn traffic opportunity, margin floor, and inventory risk into a practical growth action plan.
   - User decision supported: `Which growth actions are still worth doing after profit and inventory constraints are considered?`

The site-level GEO rule is:

- Do not present these pages as generic calculators.
- Do present them as Amazon specialist tools with clear inputs, boundaries, and operator-facing outputs.
- Every page must expose enough plain-language, indexable content for search engines and AI answer engines to understand the tool without needing to run it first.

## 3. AI Search / GEO Fit Rules

These three pages are suitable for AI search and answer-engine citation only if they expose:

1. Clear tool definition in the first screen.
2. Clear intended user and use case.
3. Clear input list.
4. Clear output list.
5. Clear limitation / non-guarantee boundary.
6. FAQ blocks that answer decision-shaped seller questions.

Best GEO framing by page type:

- Ads Audit Workbench:
  - strongest for `diagnosis`, `audit`, `what should I fix first`, `why performance is weak`
- Alexa Listing Builder:
  - strongest for `how to optimize listing for AI shopping`, `buyer question coverage`, `what copy belongs where`
- Growth & Profit Planner:
  - strongest for `growth tradeoff`, `margin floor`, `inventory-aware planning`, `is scaling still worth it`

Weak GEO patterns to avoid on all three pages:

- Tool page with only form fields and no explainer copy.
- Thin `hero + CTA + empty FAQ`.
- Generic claims like `improve sales fast`, `unlock growth`, `AI-powered optimization` without concrete decision scope.
- Keyword stuffing around `Amazon`, `AI`, `profit`, `ads`, `listing` without explaining inputs and outputs.

## 4. Ads Workbench Must Be Prioritized First

`Amazon Ads Audit Workbench` is the current flagship and should be the first page optimized for SEO/GEO.

It needs priority work on:

1. Page title:
   - Must explicitly combine `Amazon ads audit`, `PPC waste`, `TACOS`, and `growth diagnosis`.
2. Plain-language above-the-fold explanation:
   - What goes in.
   - What comes out.
   - Who this is for.
   - What it does not guarantee.
3. Indexable explainer sections:
   - Required reports.
   - Diagnosis layers.
   - Example action outputs.
   - Why this is different from a calculator.
4. FAQ block:
   - Must answer concrete Amazon seller audit questions.
5. Search-visible trust language:
   - Evidence-backed diagnosis.
   - Operator action queue.
   - Expert review compatibility.
   - No claim of autopilot ad optimization.

If only one page gets full SEO/GEO refinement first, it should be Ads Workbench.

## 5. Overpromise Boundaries For All Three Pages

Every page must explicitly avoid:

1. `Guaranteed` ranking, recommendation, conversion, TACOS, or profit outcomes.
2. Claims that the tool knows Amazon internal ranking or AI weighting.
3. Fake precision when required inputs are missing.
4. Implying fully automatic expert replacement.
5. Broad marketing language that hides real input requirements.

Safe boundary wording pattern:

- `Helps diagnose`
- `Helps prioritize`
- `Helps draft`
- `Helps structure decisions`
- `Requires user-provided inputs`
- `Should be reviewed before execution`

## 6. SEO/GEO Acceptance Checklist For Web Build Thread

When the webpage thread connects these pages, I will check:

1. SEO basics
   - Title is tool-specific, not generic.
   - Meta description matches the real decision the tool supports.
   - H1 matches the tool identity.
   - Canonical is correct.
   - Indexability is open for launch pages.

2. GEO readability
   - First screen explains tool purpose in one sentence.
   - Inputs are visible in crawlable text.
   - Outputs are visible in crawlable text.
   - Tool boundary / non-guarantee language is visible.

3. AI citation readiness
   - FAQ answers are direct, not fluffy.
   - Tool page contains at least 3-5 seller-question-shaped statements.
   - Copy can answer `what is it`, `who is it for`, `what do I input`, `what do I get`, `what does it not do`.

4. Page quality
   - Not just a blank runtime shell.
   - No internal demo wording.
   - No fake data presented as real user result.
   - No mixed Chinese/English UI leakage beyond justified tool terms.

5. Commercial credibility
   - Ads Workbench feels like a flagship workbench, not a simple widget.
   - Alexa page feels like AI-shopping listing editing, not generic copywriting.
   - Growth page feels like growth planning with profit discipline, not a shallow calculator.

## 7. Most Important Build Guidance

If the web thread has limited time, do this order:

1. Fully optimize `Amazon Ads Audit Workbench` first.
2. Launch `Alexa Listing Builder` with strong explanatory SEO/GEO copy, not only the form.
3. Launch `Amazon Growth & Profit Planner` only after its decision scope and boundary copy are clear enough to avoid being mistaken for a generic forecast toy.

The site should communicate:

`dealingnow is building Amazon specialist operator tools.`

It should not communicate:
